GSC Error Priority Calculator

Upload the Indexing → Pages CSV from Search Console (not the Performance/queries report). The tool groups issues, counts URLs, surfaces your top 10, and lets you score page value and fix effort.

Scoring model

Priority = (Impact × 0.45) + (Page value × 0.35) + (Volume × 0.20) − (Effort × 0.30)

How to use

From GSC export to sprint backlog in four steps

Everything runs in your browser. No API keys, no data sent to a server.

Which Search Console report to upload

Use the Page indexing / Pages export only. You do not add reasons manually — Google includes them in the Status column (or Reason on summary exports).

Upload this

Option A — one file (recommended)

Indexing → Pages → Not indexed → open the URL table → Export

Expected headers: URL, Last crawled, Status — the issue name is inside Status.

Option B — one issue at a time

Indexing → Pages → Why pages aren’t indexed → click a reason → Export

Combine multiple exports in Excel if you want every reason in one upload.

Option C — summary table

Issue list with Reason + Pages columns (no URL column required)

Do not upload

  • Performance → Search results (queries, clicks, impressions)
  • Experience, Core Web Vitals, or HTTPS reports
  • Sitemaps or Links exports
  • URL Inspection (single-URL check only)
  • Screaming Frog / custom CSVs unless you include url + status or reason columns yourself
  1. Export the Pages report — follow Option A or B above from Indexing → Pages.
  2. Upload the CSV — the tool groups by Status / Reason and counts URLs.
  3. Adjust scores — set page value and effort per issue; impact is suggested from the issue name (editable).
  4. Fix top 10 first — highest priority score wins your next dev sprint.
