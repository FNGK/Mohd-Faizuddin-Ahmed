# Studio CRM — setup & operations

> Live at **https://seowithfaiz.com/crm/** (noindex; access-key gated).
> Built into the same Cloudflare Worker as the website — no separate hosting.
> The old Python CMS in `cms/` is superseded (it needs its own server and
> can't run on Cloudflare Workers); kept only as reference.

## What it does
- **Leads pipeline** — every contact-form inquiry is stored automatically
  (name, email, website, market, goal, consent flags). Statuses:
  `new → contacted → replied → call → won/lost`, plus notes. Outbound
  teardown replies can be added manually ("Add lead").
- **Blog approvals** — all pipeline drafts listed with their status. Approve /
  Hold / Reject per draft; the Editor-in-Chief applies decisions on its
  Tue/Fri run (approve ⇒ publish next run, reject ⇒ `needs_revision`).
- **Integrations (CRM → ad tools)** — pipeline milestones push conversion
  signals so the ad platforms can optimize on real CRM data:
  | CRM event | GA4 (Measurement Protocol) | LinkedIn (Conversions API) | Meta (Conversions API) |
  |---|---|---|---|
  | New lead | `generate_lead` | lead event (from contact flow) | `Lead` |
  | Status → `call` | `qualify_lead` | conversion event | `Schedule` |
  | Status → `won` | `close_won` | conversion event | `Purchase` |
  Emails are SHA-256 hashed before sending; ad-platform events only fire for
  leads whose visit carried advertising consent; GA4 events require analytics
  consent. Each tool activates the moment its secret exists — until then it
  shows "pending setup" in the Integrations tab and silently no-ops.

## One-time setup (owner) — ~5 minutes total

### 1. Create the D1 database (required for lead storage)
```
npx wrangler login
npx wrangler d1 create swf-crm
npx wrangler d1 execute swf-crm --remote --file=automation/crm_schema.sql
```
Copy the `database_id` that `d1 create` prints, then un-comment and fill the
`d1_databases` block in `wrangler.jsonc`. Commit + push to deploy.

### 2. Set the access key (required for login)
```
npx wrangler secret put CRM_ACCESS_KEY
```
Pick a long passphrase — it's the CRM login. (Also add it to `APIs.env` as
`CRM_ACCESS_KEY=...` so the scheduled Editor role can call the decisions API.)

### 3. Integration secrets (each optional; add when ready)
```
npx wrangler secret put GA4_API_SECRET            # GA4 Admin → Data streams → Measurement Protocol API secrets → Create
npx wrangler secret put LINKEDIN_CONVERSIONS_TOKEN # LinkedIn OAuth token with rw_conversions (Conversions API product)
npx wrangler secret put META_PIXEL_ID              # Meta Events Manager → your dataset/pixel ID
npx wrangler secret put META_CAPI_TOKEN            # Events Manager → Settings → Conversions API → Generate token
```
Meta note: adding these enables **measurement only** (CRM → Meta CAPI). It
does not create or spend on any ads — paid activity stays a separate,
owner-approved decision per the studio guardrails.

## How the pieces talk
- UI: `crm/index.html` + `crm/crm.js` (static, brand CSS, robots-noindexed).
- API: `worker.js` → `/api/crm/*` (HMAC session cookie from `CRM_ACCESS_KEY`;
  scheduled roles authenticate with the `X-CRM-Key` header instead).
- Storage: D1 binding `CRM_DB` (tables `leads`, `blog_decisions`).
- Draft list: `blog/drafts/manifest.json`, regenerated by
  `automation/blog/generate_drafts_manifest.py` on every editorial run.
